# Contributing to HALOS

Thank you for your interest. HALOS welcomes contributions that advance the framework's clarity, coherence, and usefulness.

---

## How to Contribute

### Proposals

The primary way to shape HALOS is through **proposals**. See [proposals/README.md](proposals/README.md) for the process and [proposals/TEMPLATE.md](proposals/TEMPLATE.md) for the template.

- Proposals may add specifications, clarify principles, or suggest process changes
- All proposals are reviewed against the [principles](docs/principles.md)
- Accepted proposals are integrated into the docs

### Documentation Improvements

- **Typos and clarity** — Small fixes (grammar, formatting, broken links) can be submitted as direct pull requests
- **Substantive edits** — Changes that alter meaning or add new content should go through the proposal process

### Discussion

- Open issues for questions, ideas, or feedback
- Participate in review of others' proposals
- Help improve the proposal template or governance process via proposal

---

## Code of Conduct

HALOS is a conceptual framework, not a codebase, but we expect the same professionalism:

- Be respectful and constructive
- Critique ideas, not people
- Assume good intent
- Focus on what best serves the framework and its users

---

## What We're Looking For

Contributions that:

- **Align** with HALOS principles
- **Clarify** rather than obscure
- **Ground** abstractions in concrete use cases
- **Extend** the framework in coherent ways

We are less interested in:

- Marketing or promotional content
- Proposals that contradict core principles without justification
- Scope creep—keep proposals focused

---

## Pull Request Process

1. **Fork** the repository (or create a branch if you have write access)
2. **Make your changes** — For proposals, add a new file in `proposals/`. For small fixes, edit directly
3. **Submit a PR** with a clear description of what you changed and why
4. **Respond to review** — Maintainers or reviewers may request changes
5. **Merge** — Once approved, a maintainer will merge
